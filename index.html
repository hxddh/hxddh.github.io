Chromebook 初体验

 
几个月前买了一台 Chromebook，国内没有发售，是托论坛的人从美国带回来的。Toshiba Chromebook 2，低配版，2014年10月份发布，机子的配置是 Intel 的低功耗 CPU，2G 内存，16G SSD 硬盘，低分辨率雾面屏，合算下来是 1450 人民币，基本情况就是这样。
 


 
机子入门是有成本的，如果无法科学上网连接到 Google 的服务器，那 Chromebook 是无法激活的。登陆 Google 账户激活后，可以去领取 100G Google Drive 容量，有效期三年。系统会后台自动同步Chrome 账户下的书签、插件、应用。基本就是一台搭载 Chrome 浏览器的 PC。

 


 
Chromebook 对键盘进行了重新设计，譬如，替换 Caps Lock 键为搜索键；重定义传统的 F 系列功能键。触摸板的手势，三指切换标签页，两指回退，等等。触摸板的体验比 Windows 的要好。

适合自己的工具才值得入手，Chromebook 的优势是，续航强，轻便，对有一定技术基础且是 Chrome 重度用户的人很友好，当然大部分机器的配置比较低，除了 Pixel 二代。

如果基本不使用 Chrome 浏览器，不懂科学上网，很在意 CPU 性能，对 Web 的热情和依赖不大，那么没必要入手这类设备。
 

硬件

  • 2G 内存的配置，并不会拖累使用体验，打开 10 个左右的 Tab，不会有卡顿的感觉
  • 720P 分辨率雾面屏
  • 全塑料机身，使用低功耗 CPU，无散热风扇
  • 大于 8 小时的续航

系统

  • 底层基于 Linux，外观就是一个大号 Chrome
  • Chrome 浏览器的重度用户可以轻易上手
  • 字体和渲染不友好 ，相比 Windows 和 Mac OS，字体渲染发虚
  • 基本是基于 Internet 的设备，也有很多精美的本地 App。Chrome 应用商店已经有很多支持离线使用的 APP

适合的使用场景

  • 文字工作者，可以随时随地打开码字
  • Internet 重度使用者，随时打开浏览网页，视频
  • 不需要 PC 的本地运算性能，SSH 到服务器上干活的工程师

Hexo 与 Github 建站

 

初衷

 
  用优惠价订购了一年期的阿里云ECS，为了弥补冲动消费的后果，得利用起这台云主机。建立个人网站是其中的一项。

 

为什么选择 Hexo 与 Github

 
  追求最小的建站成本和灵活性。
  Hexo 负责生成静态网页，Github 提供了稳定的托管，阿里云ECS作为承载 Hexo 的平台。
 

 

开始建站

 

Github 侧

  • 建立分支
     建立xxx.github.io
     （其中xxx必须与github的帐户名一致）

 

ECS 侧

 

安装 Git

  • Git

 $ sudo apt-get install git-core git
 $ git config --global user.name your name
 $ git config --global user.email you email
 

  • 创建SSH Key
     
  • 与 Github 关联
     $ gedit ~/.ssh/github_rsa.pub
     将上述里面的内容copy至Github的 SSH Key 验证中

 

安装 node.js

  •  $ wget -qO- https://raw.githubusercontent.com/creationix/nvm/v0.25.4/install.sh | bash

  •  $ nvm install 0.10

  •  $ nvm alias default 0.10.40
      （很重要，否则退出终端后，重新登录后，会找不到 hexo 命令）
     

安装 Hexo

  •  $ npm install hexo-cli -g
  •  $ hexo init /blog
  •  $ cd /blog
  •  $ npm install
  •  $ npm install hexo-deployer-git --save
     （很重要，否则无法在github生成必要的文件）

 

修改 Hexo

  • 编辑 /blog 目录下的 _config.yml

deploy:
 type: git
 repository: git@github.com: xxx/xxx.github.io.git
 branch: master

  （将xxx替换成自己github的帐号）
 

建立CNAME

 
  在 /blog/source 下创建
  如果域名带二级域名 www 的话，CNAME里域名也要带www，否则会提示重定向
